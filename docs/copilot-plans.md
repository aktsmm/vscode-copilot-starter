## 🤖 エージェントモード vs Coding Agent

GitHub Copilot には **2 種類のエージェント機能** があります。混同しやすいので違いを整理します。

### Agent モード（IDE 内）

**場所**: VS Code などの IDE 内で動作

**できること**:

- 複数ファイルにまたがる編集
- ターミナルコマンドの実行（ユーザー確認あり）
- ファイル作成・削除
- コードベース全体を理解した提案

**使い方**:

1. Copilot Chat で `@workspace` を使う
2. または Chat パネルで「Agent」モードを選択

**例**:

```
@workspace このプロジェクトに認証機能を追加して
```

**特徴**:

- ✅ リアルタイムで対話しながら作業
- ✅ 変更をその場で確認・修正できる
- ✅ Free プランでも利用可能（回数制限あり）

---

### Coding Agent（GitHub 上で自律実行）

**場所**: GitHub 上（Issue や PR から起動）

**できること**:

- Issue を割り当てると **自律的にコードを書いて PR を作成**
- バックグラウンドで動作（待つ必要なし）
- PR へのフィードバックにも自動対応

**使い方**:

1. GitHub の Issue を開く
2. Copilot に Issue を割り当てる（Assignee に追加）
3. Copilot が自動でブランチ作成 → コード変更 → PR 作成

**例**:

- Issue: 「ログイン画面にパスワードリセット機能を追加」
- → Copilot が自動で実装して PR を作成

**特徴**:

- ✅ 完全自律型（人間の介入なしで作業）
- ✅ バックグラウンド実行（他の作業ができる）
- ⚠️ **Pro 以上のプランが必要**
- ⚠️ 複雑すぎるタスクは苦手（小〜中規模向け）

---

### どっちを使う？

| シナリオ                           | おすすめ         |
| ---------------------------------- | ---------------- |
| 今すぐ対話しながらコードを書きたい | **Agent モード** |
| 簡単な修正を任せて他の作業をしたい | **Coding Agent** |
| コードの説明やレビューが欲しい     | **Agent モード** |
| Issue ベースで開発を回したい       | **Coding Agent** |
| Free プランで使いたい              | **Agent モード** |

## 🔧 その他の Copilot 機能

### インライン補完（コード補完）

コードを書いている最中に自動で候補を提案する機能。

- **トリガー**: コメントを書く、コードを入力する
- **操作**: `Tab` で採用、`Esc` で却下
- **Free プラン**: 2,000 回/月

### Copilot Chat

チャット形式で Copilot と対話する機能。

- **できること**: コード生成、説明、デバッグ、リファクタリング
- **開き方**: `Ctrl + Shift + I`（または右側の Copilot アイコン）
- **Free プラン**: 50 回/月

### コードレビュー

PR に対して Copilot が自動でレビューコメントを付ける機能。

- **場所**: GitHub の PR 画面
- **対象プラン**: Pro 以上
